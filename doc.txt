Рефлексия

Какво научих:
Научих как да настройвам и конфигурирам GitHub Actions workflow за автоматизиране на CI/CD процеси. Разбрах как да използвам PHPUnit за писане на unit тестове и как да конфигурирам coverage отчети. Научих се да използвам PHP CodeSniffer за проверка на coding standards и как да го интегрирам в автоматизиран процес. Разбрах важността на разделянето на workflow-а на отделни job-ове за по-бързо изпълнение и по-добра организация. Научих как да структурирам тестове в отделни класове и как да използвам namespace-ове за по-добра организация на кода.

Кое беше най-трудно:
Най-трудното беше да намеря баланс между покритието на тестовете и тяхната изпълняемост без реална база данни. Трябваше да създам тестове, които проверяват структурата и логиката на функциите, без да разчитат на реална database връзка. Също така беше предизвикателство да конфигурирам правилно PHPUnit и PHP CodeSniffer за да работят заедно в GitHub Actions средата. Намирането на правилните версии на зависимостите и разширенията за PHP също отне известно време.

Кое може да се подобри в следваща версия:
В следваща версия бих добавил integration тестове, които използват тестова база данни за по-реалистично тестване на функциите. Бих добавил и тестове за API endpoints, използвайки PHPUnit с HTTP клиент. Може да се добави и автоматично форматиране на кода чрез PHP CS Fixer преди lint проверката. Бих подобрил и coverage отчетите, като добавя повече тестове за edge cases и error handling. Също така може да се добави cache за composer зависимостите в GitHub Actions за по-бързо изпълнение.

Кое не разбрах:
Първоначално имах трудности с разбиране на как точно GitHub Actions обработва exit codes от различните инструменти и как да конфигурирам правилно fail conditions. Също така не бях напълно сигурен как да структурирам тестовете за функции, които зависят от database връзки, без да използвам мокове или тестова база данни. Имаше и объркване относно разликата между различните PHP coding standards (PSR-1, PSR-2, PSR-12) и кой е най-подходящият за проекта.
